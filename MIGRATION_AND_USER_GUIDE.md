# 项目档案本地管理系统 V0.2：迁移与使用指南

## 1. 从 V0.1 升级

直接安装并启动 V0.2。程序会自动升级原数据库，并在同一数据目录保存
“`project_archives-升级前-日期时间.db`”副本。

原“建设单位”字段在 V0.2 中直接显示为“施工单位”，原数据不会丢失。
旧档案无法可靠推断原件与复印件数量，因此升级后默认按“1 份原件、0 份复印件”处理，
请根据实际情况编辑库存。

## 2. 项目组和子项目

新增或编辑项目时：

- 不选择上级项目：该项目作为大项目显示；
- 选择上级项目：该项目显示在大项目下；
- 建议为名称较长的子项目填写“子项目简称”，左侧树会优先显示简称。

删除大项目会同时删除它的子项目、档案和借阅历史，请先执行完整备份。

## 3. 档案库存与借阅

每条档案分别填写：

- 原件库存；
- 复印件库存；
- 可选的文字“份数描述”。

借阅时选择原件或复印件以及数量。只要还有可借库存，同一档案可以同时借给多人。
“详情/归还”中可以查看借用人、联系电话、事由、借阅份数、预计归还日期和全部历史，
并选择具体借阅记录归还。

## 4. 搜索

输入关键词后进入全局搜索结果。执行打开、绑定、借阅或查看历史等操作不会清空关键词，
可以连续处理多个模糊匹配结果。

只有点击“定位到项目”才会退出搜索并切换到对应项目。

## 5. 批量删除与回收站

点击“批量移入回收站”后，在弹窗中勾选档案，也可全选当前列表。
存在未归还借阅的档案不能删除。

点击“回收站”查看已删除档案，选择后点击“恢复所选”即可恢复。
移入回收站不会删除磁盘上的电子文件。

## 6. Excel 导入导出

### 当前项目

“Excel 导入/导出”仅处理当前项目。旧版迁移模板仍可使用；未填写库存时默认导入为
1 份原件。

### 全部项目

顶部“全部导出”生成一个工作簿，包含：

- 项目；
- 档案；
- 借阅历史；
- 导入说明。

“全部导入”会替换当前全部数据。程序会先在数据库目录生成
“`整库导入前自动备份.pambak`”。

## 7. 完整备份与换电脑

顶部“完整备份”生成 `.pambak` 文件，包含项目、档案、库存和全部借阅历史。
换电脑后安装 V0.2，点击“恢复备份”即可。

电子文件本身不会复制进备份包。换电脑时还需单独复制电子文件目录；如果盘符或目录变化，
需要重新绑定路径。

## 8. 本机应用锁

顶部点击“应用锁”设置至少 6 位密码。应用锁在下次启动时生效，连续三次错误会退出程序。
再次点击“应用锁”并验证当前密码，可以修改密码或关闭应用锁。

忘记密码时，完全退出程序，删除以下文件即可关闭应用锁，不会删除档案数据库：

```text
%LOCALAPPDATA%\ProjectArchiveManager\data\app_lock.json
```

本机应用锁用于防止普通误用，不等同于 Windows 磁盘加密或多用户权限系统。

## 9. 当前限制

- 仅支持 Windows 单机使用，不支持多人同时共享数据库；
- 不自动复制、上传或同步电子文件；
- 一条档案仍只绑定一个主电子文件；
- 安装包未进行商业代码签名时，Windows SmartScreen 可能提示未知发布者。
